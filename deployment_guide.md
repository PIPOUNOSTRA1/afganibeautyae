# دليل رفع وتشغيل متجر أفغاني بيوتي (ANABEAUTY) على خادم إنترنت حقيقي (Live Server)

يوضح هذا الدليل الخطوات اللازمة لرفع وتشغيل خادم Node.js الخلفي (Backend) والواجهة الأمامية للمتجر على الإنترنت مجاناً أو باستخدام خادم خاص (VPS).

---

## الخيار الأول: الرفع المجاني على منصة Render.com (موصى به لسهولته)

منصة **Render** هي أسهل وأسرع خيار مجاني لتشغيل خوادم Node.js.

### الخطوة 1: تجهيز كود المشروع
لقد قمنا بتجهيز ملف package.json وتحديث ملف server.js ليتوافق تلقائياً مع خوادم الويب السحابية عن طريق قراءة المنفذ العشوائي من البيئة (`process.env.PORT`).

### الخطوة 2: رفع المشروع على GitHub
1. قم بإنشاء مستودع (Repository) جديد على حسابك في GitHub (اجعله **Private** لحماية ملفات لوحة التحكم وقاعدة البيانات).
2. ارفع مجلد المشروع كاملاً إلى GitHub.

### الخطوة 3: إنشاء الخدمة على Render.com
1. قم بالتسجيل في [Render.com](https://render.com/) واربط حسابك بـ GitHub.
2. اضغط على **New** ثم اختر **Web Service**.
3. اربط الخدمة بمستودع المشروع الذي قمت برفعه.
4. اضبط الإعدادات التالية:
   * **Name**: `anabeauty-api` (أو أي اسم تفضله).
   * **Environment**: `Node`.
   * **Region**: اختر الأقرب لجمهورك (مثلاً: `Frankfurt` أو `Singapore`).
   * **Branch**: `main` (أو الفرع الافتراضي لكودك).
   * **Build Command**: `npm install` (يمكنك تركه فارغاً لعدم وجود حزم خارجية حالياً).
   * **Start Command**: `node server.js`.
   * **Instance Type**: اختر الباقة المجانية **Free**.
5. اضغط على **Create Web Service**. سيقوم الموقع ببناء الخادم وتشغيله، وسيعطيك رابطاً حقيقياً مشفراً مثل:
   `https://anabeauty-api.onrender.com`

## الخيار الثاني: التشغيل على خادم خاص (VPS) (أوبونتو Ubuntu)

إذا قمت بشراء VPS (من شركات مثل DigitalOcean أو Contabo أو Hetzner):

1. **تثبيت Node.js**:
   ```bash
   curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
   sudo apt-get install -y nodejs
   ```

2. **رفع الملفات**: انقل ملفات المشروع إلى المجلد `/var/www/anabeauty`.

3. **تشغيل الخادم في الخلفية باستمرار (PM2)**:
   ```bash
   sudo npm install -g pm2
   cd /var/www/anabeauty
   pm2 start server.js --name "anabeauty"
   pm2 startup
   pm2 save
   ```

4. **تثبيت Nginx وتمرير الطلبات (Reverse Proxy)**:
   قم بإنشاء ملف إعداد لـ Nginx يوجه المنفذ `80` (أو `443` لـ SSL) إلى المنفذ الداخلي `8080`.

## الخطوة الأخيرة: ربط المتجر بالخادم الجديد (Live API Link)

بعد الحصول على رابط الخادم الحقيقي (مثلاً: `https://anabeauty-api.onrender.com`):

1. افتح **لوحة التحكم** الخاصة بك (سواء محلياً أو بعد رفعها).
2. انتقل إلى تبويب **الإعدادات** (Settings).
3. ابحث عن قسم **إعدادات خادم الـ API (خادم الإنترنت Live Server)**.
4. ضع رابط الخادم الجديد في خانة: **رابط خادم الـ API الخارجي** واضغط على **حفظ الإعدادات**.
5. سيتم تلقائياً تحديث ملف الإعدادات؛ والآن أي طلب شراء يتم من أي جهاز أو استضافة (مثل GitHub Pages) سيتم إرساله مباشرة وحفظه بأمان في خادمك السحابي!

**ملاحظة أمنية هامّة**:
عند تفعيل الخادم السحابي، تأكد من تغيير كلمة مرور لوحة التحكم من الإعدادات فوراً لحماية بيانات عملائك وطلباتهم من التطفل.
